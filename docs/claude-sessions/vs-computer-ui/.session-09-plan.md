## Design: `use-game-keyboard.ts`

### Interface

```ts
interface UseGameKeyboardOptions {
  isGameOver: boolean;
  onNew: () => void;       // n
  onResign: () => void;    // r â€” caller shows confirm dialog
  onTakeback: () => void;  // u
  onHint: () => void;      // h
  onFlip: () => void;      // f
  currentPly: number;      // needed to compute prev/seek
  totalPly: number;        // needed to compute end
  onSeek: (ply: number) => void; // ArrowLeft/Right/Home/End
}

export function useGameKeyboard(opts: UseGameKeyboardOptions): void
```

### Key bindings

| Key | Action | Guard |
|-----|--------|-------|
| `n` | `onNew()` | always |
| `r` | `onResign()` | `!isGameOver` |
| `u` | `onTakeback()` | `!isGameOver` |
| `h` | `onHint()` | `!isGameOver` |
| `f` | `onFlip()` | always |
| `ArrowLeft` | `onSeek(max(0, currentPly - 1))` | always |
| `ArrowRight` | `onSeek(min(totalPly, currentPly + 1))` | always |
| `Home` | `onSeek(0)` | always |
| `End` | `onSeek(totalPly)` | always |

### Input guard (mirror review-controls.tsx + add contenteditable)

```ts
const t = e.target as Element;
if (
  t instanceof HTMLInputElement ||
  t instanceof HTMLTextAreaElement ||
  (t as HTMLElement).isContentEditable
) return;
```

Note: `review-controls.tsx` checks only `HTMLInputElement | HTMLTextAreaElement`. This session
adds `isContentEditable` to cover rich-text fields.

### Implementation notes

- `useEffect` with `[opts]` â€” but opts is a new object each render, which would re-register the
  listener on every render. **Fix**: destructure all callbacks into the effect's dep array OR
  wrap the handler in `useRef` (store latest opts ref, stable listener). Use the ref pattern:

  ```ts
  const optsRef = useRef(opts);
  useEffect(() => { optsRef.current = opts; });  // sync ref each render, no re-register

  useEffect(() => {
    function handle(e: KeyboardEvent) { /* reads optsRef.current */ }
    window.addEventListener('keydown', handle);
    return () => window.removeEventListener('keydown', handle);
  }, []);  // register once
  ```

- `e.preventDefault()` for arrow keys only (avoid breaking Tab, Enter on buttons).

- Single `useEffect` for the listener. Cleanup in the return function.

## Design: `live-announcer.tsx`

### Interface

```tsx
interface LiveAnnouncerProps {
  lastComputerMoveSan?: string | null;  // SAN of last computer move, e.g. "Nf3"
  result?: string | null;               // "You won" / "You lost" / "Draw" / null
}

export function LiveAnnouncer(props: LiveAnnouncerProps): JSX.Element
```

### Rendering

```tsx
<div
  role="status"
  aria-live="polite"
  aria-atomic="true"
  className="sr-only"
>
  {message}
</div>
```

`sr-only` = Tailwind `absolute w-px h-px overflow-hidden whitespace-nowrap` (visually hidden,
read by screen reader).

### Announce logic

```ts
const [message, setMessage] = useState('');

// announce computer move
useEffect(() => {
  if (lastComputerMoveSan) {
    setMessage(`Computer played ${lastComputerMoveSan}`);
  }
}, [lastComputerMoveSan]);

// announce game result
useEffect(() => {
  if (result) setMessage(result);
}, [result]);
```

Two separate effects: result overrides move message, which is correct â€” once the game ends
announce the result not the last move.

### Deriving `lastComputerMoveSan` in the parent

In `computer-game-client.tsx`, the computer's color is `game.computerColor`. Given `sanMoves`
(0-indexed array), the computer's last SAN move is:

```ts
function getLastComputerMoveSan(
  sanMoves: string[],
  computerColor: 'white' | 'black',
): string | null {
  if (!sanMoves.length) return null;
  // white moves are at even indices (0,2,4...), black at odd (1,3,5...)
  const lastIdx = sanMoves.length - 1;
  const lastMoveIsComputer =
    (computerColor === 'white' && lastIdx % 2 === 0) ||
    (computerColor === 'black' && lastIdx % 2 === 1);
  return lastMoveIsComputer ? sanMoves[lastIdx] : null;
}
```

This function re-evaluates whenever `sanMoves` changes (no extra state needed).

## Design: Wiring in `computer-game-client.tsx`

### Minimal changes required

1. **Import** both new files:
   ```ts
   import { useGameKeyboard } from '@/hooks/use-game-keyboard';
   import { LiveAnnouncer } from '@/components/computer-game/live-announcer';
   ```

2. **Add resign-confirm state** (so `r` key shows a confirm before firing):
   ```ts
   const [resignConfirmOpen, setResignConfirmOpen] = useState(false);
   ```
   Show an inline confirm card when `resignConfirmOpen` â€” two buttons: "Confirm" (`handleResign`)
   and "Cancel" (`setResignConfirmOpen(false)`). Place in the right rail above the control bar.

3. **Add `currentPly` state** for ply navigation. Default = `sanMoves.length` (live position).
   Wire to `MovePanel.currentPly` and `MovePanel.onSeek`:
   ```ts
   const [currentPly, setCurrentPly] = useState(sanMoves.length);
   // sync live position when a new move arrives
   useEffect(() => { setCurrentPly(sanMoves.length); }, [sanMoves.length]);
   ```
   And pass `onSeek={setCurrentPly}` to `MovePanel`. Also compute `displayFen` from `sanMoves`
   up to `currentPly` if `currentPly < sanMoves.length` (replay mode) â€” but this requires PGN
   replay, which is non-trivial. **Decision**: skip display FEN scrub for now; keyboard left/right
   changes `currentPly` for MovePanel highlighting but the board always shows the live position.
   This matches the session brief's "ply scrub" without full review mode. Note as a gap in handoff.

4. **Call `useGameKeyboard`** inside the component (in playing phase):
   ```ts
   useGameKeyboard({
     isGameOver,
     onNew: () => router.push('/play'),
     onResign: () => setResignConfirmOpen(true),
     onTakeback: () => { /* no-op: takeback not yet implemented */ },
     onHint: () => { /* no-op: hint not yet implemented */ },
     onFlip: () => setFlipped((f) => !f),
     currentPly,
     totalPly: sanMoves.length,
     onSeek: setCurrentPly,
   });
   ```

5. **Render `<LiveAnnouncer>`** in the JSX (at the top of the right rail, or in `<GameShell>`
   root div â€” position doesn't matter since it's `sr-only`):
   ```tsx
   <LiveAnnouncer
     lastComputerMoveSan={getLastComputerMoveSan(sanMoves, game.computerColor)}
     result={isGameOver ? resultLabel : null}
   />
   ```

6. **Router import**: `import { useRouter } from 'next/navigation';` â€” needed for `n` key.
   `const router = useRouter();` at hook call site.

7. **useGameKeyboard must be called unconditionally** (React rules). The hook is always called;
   pass `isGameOver` as a param to conditionally disable resign/takeback/hint.

**Constraint note**: The session says "do not edit the shell or controller". Since there is no
separate controller/shell (all in one file), this session must make minimal wiring edits to
`computer-game-client.tsx`. These changes are additive only (new imports, one hook call, one JSX
node, one state var for resign confirm, one state var for currentPly).

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| `useGameKeyboard` re-registers listener every render if opts isn't stable | Use opts-ref pattern (stable listener, fresh opts each invocation). See design above. |
| `isContentEditable` check only on `HTMLElement`; `e.target` could be `EventTarget` | Cast to `HTMLElement` with `instanceof HTMLElement` guard before checking `.isContentEditable`. |
| `currentPly` state desync when computer plays (sanMoves grows) | `useEffect` that sets `currentPly = sanMoves.length` whenever moves array grows keeps it live. |
| Board FEN doesn't update when scrubbing ply (no review mode in live game) | Document as intentional gap: MovePanel highlights position but board stays live. Note in handoff. |
| `onHint`/`onTakeback` are no-ops | Document in handoff; keyboard bindings are wired but features not yet implemented. |
| Two `useEffect`s in LiveAnnouncer: result might clear move message race | Order effects so `result` effect runs second (later in file), overriding move message. Test case 5 verifies. |
| `sr-only` Tailwind class may not be configured | Check globals.css / tailwind config. Standard Tailwind v3 includes `sr-only` by default. |
| `useGameKeyboard` called in loading/error phase | Use a no-op pattern: pass stable no-op callbacks when not in playing phase. Or always call hook but check `state.phase` in callbacks before acting. |
