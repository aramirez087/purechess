## Design Decisions and Rationale

### 1. `use-game-keyboard.ts` — callback-prop design

**Why callbacks not a zustand store:** There is no `use-computer-game.ts` controller. Callbacks
match the existing style (`handleResign`, `setFlipped`, `handleMove` are all plain functions in
the client). A callback-prop hook is also trivially testable without mocking stores.

**Interface:**
```ts
export interface UseGameKeyboardOptions {
  isGameOver: boolean;
  isComputerThinking: boolean;  // block u/h/r while bot computes
  currentPly: number;           // for seek clamping
  totalPly: number;
  onHint?: () => void;          // h key
  onTakeback?: () => void;      // u key
  onResign?: () => void;        // r key → consumer shows confirm dialog
  onFlip?: () => void;          // f key
  onNew?: () => void;           // n key → navigate to /play
  onSeek?: (ply: number) => void; // ArrowLeft/Right/Home/End
}

export function useGameKeyboard(opts: UseGameKeyboardOptions): void
```

**Key bindings:**
| Key | Action | Guard |
|-----|--------|-------|
| `h` | `onHint?.()` | skip if `isGameOver || isComputerThinking` |
| `u` | `onTakeback?.()` | skip if `isGameOver || isComputerThinking` |
| `r` | `onResign?.()` | skip if `isGameOver || isComputerThinking` |
| `f` | `onFlip?.()` | always (can flip after game too) |
| `n` | `onNew?.()` | always (works before and after game) |
| `ArrowLeft` | `onSeek?.(max(0, currentPly-1))` | prevent default (stop page scroll) |
| `ArrowRight` | `onSeek?.(min(totalPly, currentPly+1))` | prevent default |
| `Home` | `onSeek?.(0)` | prevent default |
| `End` | `onSeek?.(totalPly)` | prevent default |

**Text-field guard (mirrors review-controls.tsx):**
```ts
const t = e.target as Element;
if (
  t instanceof HTMLInputElement ||
  t instanceof HTMLTextAreaElement ||
  t.isContentEditable
) return;
```
This guards ALL bindings, including arrow keys.

**Board-focus guard for arrow keys:** When a board square (`[role=gridcell]`) has focus,
arrow keys navigate squares in `chessboard.tsx`. To avoid double-firing, check:
```ts
if (isArrowKey && t.closest('[role="grid"]')) return;
```
This lets the board handle its own arrow navigation while the global seek fires when focus
is elsewhere (move list, rail, nothing).

**Stable callbacks via latest-ref pattern:** Wrap each callback in a `useRef` updated
inside a layout effect so the `keydown` listener never needs to be torn down on prop change.
This avoids the `useEffect` re-subscribe churn that `review-controls.tsx` has (its deps
array lists 4 callbacks; every re-render with an inline function creates a new listener).

```ts
const optsRef = useRef(opts);
useLayoutEffect(() => { optsRef.current = opts; });

useEffect(() => {
  function handleKey(e: KeyboardEvent) { /* reads optsRef.current */ }
  window.addEventListener('keydown', handleKey);
  return () => window.removeEventListener('keydown', handleKey);
}, []); // intentionally empty — stable via ref
```

### 2. `live-announcer.tsx` — aria-live polite region

**Props:**
```ts
export interface LiveAnnouncerProps {
  /** Last computer move in SAN notation, e.g. "Nf3". Null before first computer move. */
  lastComputerMoveSan: string | null;
  /** Human-readable result, e.g. "You won", "You lost", "Draw". Null while active. */
  gameResult: string | null;
}
```

**Caller responsibility:** The consumer computes `lastComputerMoveSan` from `parsePgnMoves`
and the game's `computerColor`. This is done in `computer-game-client.tsx` (wiring step).

**Announcement logic:**
```ts
const [announcement, setAnnouncement] = useState('');
const prevSanRef = useRef<string | null>(null);

useEffect(() => {
  const sanChanged = lastComputerMoveSan && lastComputerMoveSan !== prevSanRef.current;
  if (sanChanged) {
    prevSanRef.current = lastComputerMoveSan;
    const msg = gameResult
      ? `Computer played ${lastComputerMoveSan}. ${gameResult}.`
      : `Computer played ${lastComputerMoveSan}.`;
    setAnnouncement(msg);
  } else if (gameResult) {
    setAnnouncement(`Game over. ${gameResult}.`);
  }
}, [lastComputerMoveSan, gameResult]);
```

Covers three cases:
- Computer makes a mid-game move → "Computer played Nf3."
- Computer's move ends the game → "Computer played Nxf7+. You lost."
- Human resigns (no new computer SAN) → "Game over. You lost."

**Rendering:**
```tsx
<div
  role="status"
  aria-live="polite"
  aria-atomic="true"
  className="sr-only"
>
  {announcement}
</div>
```

`role="status"` is equivalent to `aria-live="polite"` + `aria-atomic` per ARIA spec, but
naming both explicitly maximizes screen reader compatibility. `sr-only` = Tailwind visually
hidden (absolute, 1px clip, overflow hidden).

**Why `sr-only` not `hidden`:** `hidden` or `display:none` suppresses aria-live entirely
in some browsers. The Tailwind `sr-only` class keeps the element in the accessibility tree
but off-screen.

Uses `renderHook` + `fireEvent.keyDown(window, { key })` for the hook, and `render` +
`screen.getByRole('status')` for the announcer. No component wrapper needed for the hook test
because `renderHook` provides the React context.

**Test cases (covering the 4 required by session instructions):**

```

  ✓ announces "Game over. Draw." when only gameResult changes (no new computer move)
```

## Risks and Mitigations

| Risk | Mitigation |
|------|------------|
| `fireEvent.keyDown(window, ...)` may not work in jsdom | Use `window.dispatchEvent(new KeyboardEvent('keydown', { key: 'h', bubbles: true }))` as fallback |
| `renderHook` not available | It is in `@testing-library/react` v14, which is already installed (used in other tests) |
| `isContentEditable` check throws on non-element `e.target` | Cast via `(e.target as Element | null)?.isContentEditable` |
| Board arrow key conflict | Handled by `closest('[role="grid"]')` guard |
| Same SAN announced twice (promotions from same square) | `prevSanRef` comparison uses the SAN string identity — distinct moves will always differ |
| `sr-only` not defined in Tailwind config | It's a core Tailwind utility (`.sr-only`), no config needed |
| Integration skipped means no live a11y improvement | Document integration wiring clearly in handoff so next session can wire in ≤10 min |

const lastComputerMoveSan: string | null = (() => {
