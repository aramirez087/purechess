## Design decisions

### Task 1 — Level button visual overhaul

**Problem:** Current selected state (`bg-brass/10 border-brass/60`) is too subtle — the "active" and "inactive" treatments read as nearly the same on a dark background.

**Solution:**
- Selected: `bg-brass text-background font-semibold ring-2 ring-brass ring-offset-1 ring-offset-background` — solid gold fill, dark text, ring halo. Clearly unmistakable.
- Inactive: `bg-raised/40 border-border/60 text-muted-foreground` — dimmer than selected. `hover:bg-raised hover:text-foreground` for affordance.
- Hit target: `h-11` (was `h-10`).
- Visible label: the existing `Level {level} · {LEVEL_LABELS[level]}` span already serves this role — keep it, style with `text-brass font-medium`.
- Keep `aria-pressed` on every button.
- Add `CheckIcon` (lucide, already available) rendered inside the selected button only. Size `h-3 w-3` inline before the number, or overlay in corner.

**Rationale:** Gold fill on dark background is unambiguous. Avoids a check-only approach (hard to see at small size); fill + ring is the standard "selected tile" pattern in premium dark UIs.

### Task 2 — Time-control picker

**State:** `timePreset: string | 'untimed'` — key into `TIME_CONTROL_PRESETS` or the sentinel `'untimed'`.

**Presets to show (curated from TIME_CONTROL_PRESETS):**
| Label | Key | seconds | inc |
|-------|-----|---------|-----|
| Untimed | `'untimed'` | 600 (dummy) | 0 |
| Bullet 1\|0 | `'1+0'` | 60 | 0 |
| Blitz 3\|2 | `'3+2'` | 180 | 2 |
| Blitz 5\|0 | `'5+0'` | 300 | 0 |
| Rapid 10\|0 | `'10+0'` | 600 | 0 |

Five buttons in a single row (`grid-cols-5`), same `aria-pressed` pattern and same gold-fill active treatment as levels.

**Payload mapping:**
- `untimed` → `timeControlSeconds: 600, incrementSeconds: 0` (current behavior, backward-compat)
- any preset key → `timeControlSeconds: preset.initialSeconds, incrementSeconds: preset.incrementSeconds`

**Default:** `'untimed'` (preserves existing default behavior).

**Header copy:** Replace `"Untimed · 8 difficulty levels"` with a derived string: `"{timeLabel} · {strengthDescription}"` updated reactively.

### Task 3 — Strength mode, style/blunder, think-time

**Strength mode toggle state:** `strengthMode: 'level' | 'elo'`

Rendered as two pill/tab buttons (`aria-pressed`) above the level grid or ELO slider:
- `Level` → shows the 1–8 grid (existing UI)
- `ELO Target` → hides the grid, shows a number input + range slider (range 600–2800, step 50)

**ELO state:** `eloTarget: number` default `1500`

**Payload mapping:**
- `strengthMode === 'level'` → send `level`, omit `eloTarget`
- `strengthMode === 'elo'` → send `eloTarget`, still send `level: 4` as a fallback (API requires `level` field; `eloTarget` overrides in the service)

**Human-like play (styleBlunderCp):**
State: `humanLike: boolean` default `false`
When true → `styleBlunderCp: 50` (±50cp noise window, reasonable default).
Rendered as a Radix `Switch` with label "Human-like play" and sublabel "Adds occasional inaccuracies".

**Think-time (thinkTimeMs):**
State: `thinkTime: 'auto' | 'fast' | 'normal' | 'slow'` default `'auto'`
Rendered as a 4-button row (same pill pattern):
| Label | thinkTimeMs value |
|-------|-------------------|
| Auto | `undefined` (omit) |
| Fast | 300 |
| Normal | 1000 |
| Slow | 2500 |

Use a `<select>` or button row — prefer button row for visual consistency with existing pill pickers.

**Payload construction (all cases):**
```ts
const payload: CreateComputerGameDto = {
  level: strengthMode === 'elo' ? 4 : level,
  color,
  timeControlSeconds: resolvedTimeControl.initialSeconds,
  incrementSeconds: resolvedTimeControl.incrementSeconds,
  ...(strengthMode === 'elo' && { eloTarget }),
  ...(humanLike && { styleBlunderCp: 50 }),
  ...(thinkTime !== 'auto' && { thinkTimeMs: THINK_TIME_MAP[thinkTime] }),
};
```

### Task 4 — Backward compatibility

Default state produces identical payload to current code:
- `level: 4, color: 'random', timeControlSeconds: 600, incrementSeconds: 0`
- No `eloTarget`, no `styleBlunderCp`, no `thinkTimeMs`

The `UNTIMED_SECONDS` / `UNTIMED_INCREMENT` constants can be replaced by the 'untimed' preset's resolved values (600/0) — same numbers.
