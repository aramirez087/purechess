## Design Decisions

### 1. ECO data shape (`eco.ts`)

```ts
export interface EcoEntry {
  code: string;   // "B20"
  name: string;   // "Sicilian Defence"
  moves: string;  // SAN space-separated, e.g. "e4 c5"
}
export const ECO_OPENINGS: EcoEntry[] = [ ... ]
```

**Rationale:** Store only moves (not pre-computed FEN) to keep the data file small and
human-readable. FENs are computed at runtime via chess.js (already a dep) in `index.ts`.
~50 curated entries covering the most-played openings by ECO section (A/B/C/D/E).

Curated coverage:
- A: English, Reti, Bird, King's Indian Attack
- B: Sicilian (main lines: Najdorf, Dragon, Classical, Scheveningen), Pirc, Caro-Kann, French (Advance)
- C: Ruy Lopez, Italian, King's Gambit, French (Classical/Winawer), Petrov
- D: Queen's Gambit (Accepted/Declined), Slav, London, Colle
- E: King's Indian, Nimzo-Indian, Grunfeld, Catalan, Dutch

### 2. `index.ts` helpers

```ts
// FEN after applying SAN moves (throws propagated to callers)
export function applyMoves(sanMoves: string): string

// Returns FEN for a given ECO entry; memoised via a Map<code, string>
export function getEcoFen(entry: EcoEntry): string

// Case-insensitive substring search on name and code
export function lookupByName(query: string): EcoEntry | undefined

// Normalize board position (first 4 FEN fields, drop move clocks)
// and compare against all known ECO FENs
export function lookupByFen(fen: string): EcoEntry | undefined

// Picks a random ECO entry and returns { fen, name, code }
export function randomOpening(): { fen: string; name: string; code: string }

// Returns true iff chess.js can load the FEN without throwing
export function isValidFen(fen: string): boolean
```

**FEN normalization for `lookupByFen`:** Compare only the first 4 space-separated tokens
(board, side-to-move, castling, en-passant) — drop halfmove/fullmove clocks so a position
reached via transposition still matches.

**Memoization:** Use a `Map<string, string>` keyed on ECO code. Computed on first access.
Avoids re-running chess.js for every render of the picker list.

### 3. `OpeningPicker` component

**Props:**
```ts
interface OpeningPickerProps {
  onSelect: (fen: string, name: string, code: string) => void;
}
```

**UI pattern:** Accessible combobox built with:
- `<input role="combobox" aria-expanded aria-autocomplete="list" aria-controls="opening-listbox">`
- Absolute-positioned list `<ul role="listbox" id="opening-listbox">` rendered when `open`
- "Random Opening" button below the input (always visible)
- Selected state: show ECO badge (`code — name`) below input; "Clear" button to reset

**No new deps** — no cmdk/popover needed. Radix dropdown-menu is awkward for a searchable
input pattern; a hand-rolled combobox is cleaner and the list is short enough.

**Keyboard nav:**
- `ArrowDown` / `ArrowUp` → move `activeIndex` through filtered results
- `Enter` → select `activeIndex` item
- `Escape` → close list
- `scrollIntoView` the active item via `el?.scrollIntoView?.({ block: 'nearest' })` (jsdom-safe)

**Filtering:** Filter `ECO_OPENINGS` by `name.toLowerCase().includes(query)` or
`code.toLowerCase().startsWith(query)`. Show max 40 results when no query (full list would
be 50 items, still manageable).

**Styling:** Match `computer-game-setup.tsx` palette — `border-border/70 bg-raised/40`,
gold (`text-brass`) for selected state, same button classes as existing setup form.

### 4. `FenSetupBoard` component

**Props:**
```ts
interface FenSetupBoardProps {
  level: CreateComputerGameDto['level'];
  color: CreateComputerGameDto['color'];
  timeControlSeconds: number;
  incrementSeconds: number;
  onGameCreated: (gameId: string) => void;
  onCancel: () => void;
}
```

**Internal state:**
- `fen: string` — current input value
- `valid: boolean` — computed from `isValidFen(fen)` whenever fen changes
- `error: string | null` — inline error message
- `isPending: boolean` — submit in-flight

**FEN validation:** Derived (not stored) — recompute on each render from `fen` using
`isValidFen`. Only show error after the user has typed something (blur or non-empty input).

**Chessboard preview:** Render `<Chessboard position={fen} readOnly />` when `valid`.
Wrap the whole component in `<BoardSettingsProvider>` because Chessboard needs the
board-context.

**Submit:** Calls `createComputerGameFromFen({ fen, level, color, timeControlSeconds, incrementSeconds })`
and on success calls `onGameCreated(result.gameId)`.

**Orientation:** Always `'white'` for the preview (static, doesn't depend on chosen color).

### 5. Vitest tests (`openings.test.ts`)

No DOM needed; pure `vitest` (no `@testing-library`). Run in Node environment.

Tests:
1. `lookupByName('Sicilian')` → code starts with 'B', name includes 'Sicilian'
2. `lookupByName('ruy lopez')` → name includes 'Ruy Lopez' (case-insensitive)
3. `lookupByName('zzzznotfound')` → `undefined`
4. `getEcoFen(ECO_OPENINGS[0])` → chess.js accepts FEN without throwing (i.e., `isValidFen` returns true)
5. `randomOpening()` → result has string `fen`, `name`, `code`; `isValidFen(fen)` is true
6. `isValidFen('rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq - 0 1')` → true
7. `isValidFen('not a fen')` → false
8. `isValidFen('rnbqkbnr/pppppppp/8/8/8/8/PPPPPPPP/RNBQKBNR w KQkq')` → false (missing fields)

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| chess.js ESM in vitest | Existing tests use it (see `material.test.ts`, `replay.ts`). Vitest config handles it. No change needed. |
| `scrollIntoView` undefined in jsdom | Call via `el?.scrollIntoView?.({ block: 'nearest' })` — safe call pattern from cerebrum. |
| FEN with extra whitespace from paste | `fen.trim()` before passing to chess.js. |
| `lookupByFen` false negatives | Normalize to 4-token prefix before comparison. |
| `BoardSettingsProvider` missing for Chessboard | Wrap `FenSetupBoard` root in `<BoardSettingsProvider>`. |
| `docs/roadmap/vs-computer-ui/` not yet created | Create dir implicitly when writing session-07-handoff.md (Write tool creates parent dirs). |
| Chess.js `new Chess(fen)` may throw on partial FEN | Wrap in try/catch and return false from `isValidFen`. |
| Curly quotes in code | Write all string literals with ASCII `'` only. |
| ECO moves invalid in chess.js | Manually verify moves in eco.ts data against the starting position when writing the data. |
