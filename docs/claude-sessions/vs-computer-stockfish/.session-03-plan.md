## Step-by-Step Order of Operations

### Step 1 — `apps/web/src/lib/api/computer-games.ts` (new file)

Pattern: copy `apiFetch` from `reports.ts` (identical implementation, no shared helper yet — session scope forbids adding dependencies).

```ts
import type { CreateComputerGameDto, ComputerGameStateDto } from 'shared';

const API = process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:4000';

async function apiFetch<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${API}/api${path}`, {
    credentials: 'include',
    headers: { 'Content-Type': 'application/json' },
    ...init,
  });
  if (!res.ok) {
    const body = await res.json().catch(() => ({}));
    throw Object.assign(
      new Error((body as { message?: string }).message ?? res.statusText),
      { status: res.status },
    );
  }
  return res.json() as Promise<T>;
}

export function createComputerGame(dto: CreateComputerGameDto): Promise<ComputerGameStateDto> {
  return apiFetch('/computer-games', { method: 'POST', body: JSON.stringify(dto) });
}
```

**Import path for shared types:** Check `tsconfig.json` path aliases for `shared` vs `@purechess/shared` vs relative import. Use whatever pattern `use-invite.ts` or other hooks use for shared types. (No hook currently imports from shared; the DTO file is `packages/shared/src/dto/computer-game.dto.ts` re-exported from `packages/shared/src/index.ts`.) Use the workspace package name as configured in `package.json` of `packages/shared`.

**LEVEL_LABELS** for accessibility tooltips:
```ts
const LEVEL_LABELS: Record<number, string> = {
  1: 'Beginner', 2: 'Beginner+', 3: 'Easy',
  4: 'Intermediate', 5: 'Intermediate+', 6: 'Advanced',
  7: 'Expert', 8: 'Master',
```

**Layout** (mirrors `InviteCreate` structure — `Card > CardHeader + CardContent`):
1. Level picker: `grid grid-cols-4 gap-2` — 8 `Button` components (size="sm"), `variant="default"` when selected, `variant="outline"` otherwise. Show `LEVEL_LABELS[level]` as `text-xs text-muted-foreground` below grid.
2. Color picker: same 3-button row as `InviteCreate`.
3. Time control picker: 3-button row (Bullet/Blitz/Rapid).
4. Action row: "Start Game" `Button` (disabled + shows spinner text while `isPending`) + "Back" ghost button calling `onCancel`.
5. Error display: `<p className="text-sm text-destructive">` when `error` is set.

**handleStart:**
```ts
async function handleStart() {
  setIsPending(true);
  setError(null);
  try {
    const tc = TIME_CONTROLS[tcIndex];
    const result = await createComputerGame({
      level: level as CreateComputerGameDto['level'],
      color,
      timeControlSeconds: tc.seconds,
      incrementSeconds: tc.increment,
    });
    onGameCreated(result.gameId);
  } catch (e) {
    setError(e instanceof Error ? e.message : 'Failed to start game. Please try again.');
  } finally {
    setIsPending(false);
  }
}
```

No React Query here — single fire-and-forget mutation. Keeps it simple, consistent with the session scope.

## Design Decisions & Rationale

| Decision | Rationale |
|----------|-----------|
| New `api/computer-games.ts` file, not modifying `api.ts` | `api.ts` is server-only (`cookies()` from `next/headers`). Client components must not import it. The `reports.ts` precedent establishes the pattern. |
| Local `apiFetch` copy in each api module | No shared client helper exists yet. DRY refactor is out of session scope. Three files with the same ~8-line helper is acceptable; premature abstraction avoided. |
| No React Query `useMutation` in `ComputerGameSetup` | Fire-and-forget pattern. The mutation is not re-used elsewhere, has no cache invalidation needs, and adding a hook file is out of scope. Local `useState` + async handler is simpler. |
| Default level 4 (Intermediate) | Middle of the range — best first impression for typical player. |
| Default color "random" | Matches `InviteCreate` precedent. |
| Default time control Blitz 3+2 | Session spec lists "Blitz (3+2)" as the middle option; session spec index 1 maps to it. |
| `router.push('/computer-game/' + gameId)` on success | Session 04+ will implement that route. The navigation will 404 until then — acceptable, handoff documents it. |
| No loading skeleton | Session scope does not include loading states beyond a disabled button. |

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Shared package import alias unknown | Grep before writing: `grep -r "from '.*shared" apps/web/src --include="*.ts" --include="*.tsx" -l` then inspect one match. Fall back to `../../../../../../packages/shared/src` relative path only if no alias exists — unlikely given monorepo setup. |
| `ComputerGameStateDto` shape mismatch with actual API response | Session 02 hasn't run yet; the API endpoint may not exist at test time. The TypeScript type is locked by Session 01 DTO. Any runtime 404 is expected until Session 02 ships — document in handoff. |
| `router` not available server-side | `play-page-client.tsx` is already `'use client'`, so `useRouter` works. |
| PostHog import missing | Already imported in `play-page-client.tsx` line 6. |
| `useRouter` not currently imported in `play-page-client.tsx` | Must add `import { useRouter } from 'next/navigation';` — easy, mirrors `invite-create.tsx`. |
| Level type `1|2|...|8` vs `number` | `CreateComputerGameDto.level` is the union type. The `level` state is `number`; cast with `level as CreateComputerGameDto['level']` — safe because state is only set from the 1–8 button array. |
