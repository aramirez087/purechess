## Step-by-Step Order of Operations

### Step 1 ŌĆö Add `computerColor` to shared DTO

`packages/shared/src/dto/computer-game.dto.ts`

Add `computerColor: 'white' | 'black'` to `ComputerGameStateDto`.

**Rationale:** The client needs to orient the board (flip if user plays black). The DTO
currently has no color field. This is an additive change; no existing consumer reads this
field, so it is non-breaking. The DB stores `computerColor` on the Game row ŌĆö we just
need to thread it through.

```typescript
export interface ComputerGameStateDto {
  gameId: string;
  fen: string;
  pgn: string;
  status: string;
  computerColor: 'white' | 'black';   // ŌåÉ ADD
  lastComputerMove: string | null;
  result: string | null;
  resultReason: string | null;
}
```

if (dto.move === 'resign') {
  const userIsWhite = game.whiteUserId === userId;
  // If user resigns, computer wins
  const result = userIsWhite
    ? GameResult.BlackWins   // user=white resigned ŌåÆ black (computer) wins
    : GameResult.WhiteWins;  // user=black resigned ŌåÆ white (computer) wins

    where: { id: gameId },
    data: {
      status: 'completed',
      result: result as unknown as PrismaGameResult,
      resultReason: reason as unknown as GameResultReason,
      endedAt: new Date(nowMs),
    },
  });

  return toStateDto(
    gameId,
    game.finalFen ?? STARTING_FEN,
    game.pgn,
    'completed',
    game.computerColor as 'white' | 'black',
    game.lastComputerMove ?? null,
    result,
    reason,

Import `GameTermination` from `@purchess/shared` (already imported `GameResult`).

### Step 3 ŌĆö Add client API helpers

`apps/web/src/lib/api/computer-games.ts`

Add two exports (reusing existing `apiFetch`):

```typescript
export function getComputerGame(gameId: string): Promise<ComputerGameStateDto> {
  return apiFetch(`/computer-games/${gameId}`);
}

export function submitComputerMove(
  gameId: string,
  move: string,
): Promise<ComputerGameStateDto> {
  return apiFetch(`/computer-games/${gameId}/move`, {
    method: 'POST',
    body: JSON.stringify({ move }),
  });
}
```

### Step 4 ŌĆö Server component: `page.tsx`

`apps/web/src/app/computer-game/[gameId]/page.tsx`

Thin server component ŌĆö just passes `params.gameId` down. No server-side data fetch here
because the game requires auth cookies only the client can send with `credentials: 'include'`.

```typescript
import { ComputerGameClient } from './computer-game-client';

interface Props {
  params: { gameId: string };
}

export default function ComputerGamePage({ params }: Props) {
  return <ComputerGameClient gameId={params.gameId} />;
}
```

**Rationale for no SSR fetch:** `getComputerGame` is a client-side helper using
`credentials: 'include'`. The server component cannot forward the session cookie in this
pattern without importing `api.ts` (server-only) and threading cookies manually ŌĆö that
complexity is out of session scope. Initial render will show a loading state for one RTT,
which is acceptable for a game board.

### Step 5 ŌĆö Client component: `computer-game-client.tsx`

`apps/web/src/app/computer-game/[gameId]/computer-game-client.tsx`

**State shape:**
```typescript
type PageState =
  | { phase: 'loading' }
  | { phase: 'error'; message: string }
  | { phase: 'playing'; game: ComputerGameStateDto; submitting: boolean };
```

**On mount:** `useEffect(() => { getComputerGame(gameId).then(...) }, [gameId])`

**Board orientation:**
```typescript
const orientation = game.computerColor === 'white' ? 'black' : 'white';
// computer plays white ŌåÆ board shows black side up (user is black)
```

**`readOnly` prop:** `true` when `game.status === 'completed' || submitting`
- When `submitting` is true (computer's turn being computed), show a spinner overlay on
  the board or a "Computer is thinkingŌĆ”" status message.

**`onMove` callback:**
```typescript
async function handleMove(intent: MoveIntent) {
  const uci = intent.from + intent.to + (intent.promotion ?? '');
  setState(s => s.phase === 'playing' ? { ...s, submitting: true } : s);
  try {
    const next = await submitComputerMove(gameId, uci);
    setState({ phase: 'playing', game: next, submitting: false });
  } catch (err) {
    // On error: reset submitting, surface error inline
    setState(s => s.phase === 'playing' ? { ...s, submitting: false } : s);
    setError((err as Error).message);
  }
}
```

**`lastMove` prop:** Parse `game.lastComputerMove` (UCI string like `"e7e5"`) into
`{ from: 'e7', to: 'e5' }` for board highlighting.

**Result banner** (shown when `status === 'completed'`):
- Map `result` + `computerColor` to "You won" / "You lost" / "Draw"
- Show `resultReason` with a human-readable label map

```typescript
function getResultLabel(
  result: string | null,
  computerColor: 'white' | 'black',
): string {
  if (!result) return '';
  if (result === 'draw') return 'Draw';
  const computerWins =
    (result === 'white_wins' && computerColor === 'white') ||
    (result === 'black_wins' && computerColor === 'black');
  return computerWins ? 'You lost' : 'You won';
}
```

**Resign button:**
- Shown only when `status === 'active'` and not `submitting`
- Calls `submitComputerMove(gameId, 'resign')`
- On resolve: update state; on reject: show inline error

**Move list (sidebar):**
- Parse `game.pgn` to extract SAN tokens, display in a scrollable list
- Simple regex split: `pgn.split(/\d+\.\s+/).filter(Boolean)` then tokenize pairs
- `useEffect` to auto-scroll a `ref` on pgn change

**Layout:** Two-column on desktop (board | sidebar), single column on mobile.

## Design Decisions & Rationale

| Decision | Rationale |
|----------|-----------|
| Add `computerColor` to DTO | Only reliable way to orient the board; additive field, non-breaking |
| Client-only fetch (no SSR) | Session scope; avoids duplicating auth cookie forwarding logic |
| `submitting` flag disables board via `readOnly` | Prevents double-submit and move during computer turn; cleanest path through `Chessboard` API |
| Resign sends `{ move: 'resign' }` to existing endpoint | Reuses auth/ownership checks in `submitMove`; no new endpoint needed |
| `GameTermination.Resignation` string value `'resignation'` | Matches `DbGameResultReason.resignation` ŌĆö cast is safe |
| PGN move list from DTO | PGN is already built and returned by service; no need for a separate moves array |

---

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| `game.computerColor` nullable in Prisma type | Add non-null assertion with comment; invariant holds for all `isVsComputer` games |
| PGN parsing edge cases (annotations, clock times) | Use `chess.js` `Chess` object to load PGN and extract `history({ verbose: false })` for clean SAN tokens ŌĆö more robust than regex |
| Race: user clicks move while `submitting=true` | `readOnly={submitting}` prevents board interaction; no extra guard needed |
| `submitComputerMove('resign')` fails mid-render | Catch in handler, show error message, do not update state |
| `computerColor` field missing in older game records | All games created after Session 02 migration have this field; MVP scope is new games only |
