## 2. Design decisions + rationale

**D1 — Single shared Worker, serialized commands.** Keep existing lazy `workerPromise` + `uciok`
handshake. UCI is stateful (one search at a time). Add an internal **command queue / single
in-flight job** so analyze/getHint/getBestMove don't interleave `info`/`bestmove` lines on the
shared worker. Simplest correct form: a module-level `Promise` chain (`currentJob`) that each
public call awaits before attaching its own message listener and issuing `go`.

**D2 — `info` line parser.** Pure function `parseInfoLine(line): Partial<EngineEval> & {multipv?:number}`.
Parse tokens: `depth <n>`, `score cp <n>` / `score mate <n>`, `multipv <n>`, `pv <m1> <m2> ...`.
`pv` is last — consume rest of tokens. `bestmove = pv[0]`. Keep latest info per `multipv` index in a
`Map<number, EngineEval>`. On `bestmove`, resolve with the stored line(s).
- Score POV: UCI reports score from side-to-move POV already → matches `EngineEval` contract. No flip.
- Lowerbound/upperbound tokens: ignore (still parse cp/mate).

**D3 — `analyze(fen, options)`.** Resolves final `EngineEval` (multipv=1 line). Flow:
1. await worker + queue slot.
2. Apply options: `setoption name MultiPV value <multiPv|1>`, strength (D4), `position fen <fen>`,
   `go movetime <movetimeMs|1000>`.
3. Collect `info` lines into `Map<multipv, EngineEval>`.
4. On `bestmove`, resolve. Return the `multipv===1` line (or best by depth). Expose all lines via
   internal collection used by getHint/blunder (see D5). **Reset MultiPV to 1 after** non-default
   use so shared worker state doesn't leak into next caller (setoption before each go covers it).

Return type stays `Promise<EngineEval>` per shared contract. Top-N lines needed by blunder logic →
keep them in a private `analyzeLines(fen, options): Promise<EngineEval[]>` helper; `analyze` returns
`lines[0]`.

**D4 — Strength: two modes.**
- ELO mode (when `options.eloTarget` set): `setoption name UCI_LimitStrength value true` +
  `setoption name UCI_Elo value <n>`. Clamp to engine range (typical 1320–3190); clamp + document.
- Skill mode (default / when `skill` or `level` given): `setoption name UCI_LimitStrength value false`
  + `setoption name Skill Level value <skill>`. Reuse `UCI_SKILL` 8-bucket map for level→skill in
  `getBestMove`/`getHint`. **Always emit BOTH the LimitStrength toggle and the relevant value** each
  call so the shared worker can't retain a stale ELO cap from a prior analyze.

**D5 — Blunder / style knob (deterministic).** When `options.blunderCp` set AND `multiPv>1`:
1. analyzeLines with MultiPV ≥ 2 (force `multiPv = max(multiPv, 2)`, cap 3).
2. topCp = line[0] score in cp (mate → large sentinel ±100000, signed).
3. Eligible = lines whose cp within `[topCp - blunderCp, topCp]` (side-to-move POV; only "not better
   than best, not worse than window").
4. Pick deterministically: index = `seededPick(eligible.length, deterministicSeed ?? 0)` — small LCG /
   mulberry32 on the seed. Deterministic given seed (no `Math.random`). Default seed 0 → first
   eligible (= best) so behavior stable when seed omitted.
This selection used by `getHint` and an internal "human move" path; `analyze` itself still returns the
true best line (analysis ≠ move-choice). Document that distinction in handoff.

**D6 — think-time.** `movetimeMs` option threads into `go movetime`. Default 1000 (unchanged).
`getBestMove(fen, level, movetimeMs=1000)` keeps its 3rd positional arg.

**D7 — cancellation.** Module-level `cancel()` sends `stop` to the worker and rejects/settles the
in-flight job. Implementation: store the active job's `reject` (+ cleanup) in a module ref
`activeJob`. `cancel()` → if active: `worker.postMessage('stop')`, remove listener, clear timeout,
reject with `EngineCancelledError` (typed). Stockfish will still emit a `bestmove` after `stop`; the
listener is already removed so it's ignored, and the queue advances. Per-call cancel token: also
accept optional `AbortSignal` later if needed — for this session a module-level `cancel()` matching
the handoff signature `cancel(): void` is sufficient. (Note: shared worker = one in-flight job, so a
single `cancel()` is unambiguous.)

**D8 — warm-up + timeout error.**
- `warmUp(): Promise<void>` → `await getWorker(); return;` (triggers handshake, resolves on `uciok`).
- Export `ready` access via warmUp (no separate promise needed; getWorker already memoizes).
- Replace silent timeout: define `class EngineTimeoutError extends Error` (and
  `EngineCancelledError extends Error`) with discriminable `name`. Timeout path rejects with
  `new EngineTimeoutError(...)` instead of `new Error('Stockfish timeout...')`. Export the class so
  callers can `instanceof`/catch. Keep `BESTMOVE_TIMEOUT_MS = 10_000` default; make per-call timeout
  = `max(movetimeMs + buffer, BESTMOVE_TIMEOUT_MS)` so long think-times don't false-timeout.

**D9 — DTO file: prefer NOT to modify.** `EngineEval`/`EngineAnalysisOptions` already cover all needs
(movetimeMs, multiPv, eloTarget, skill, blunderCp, deterministicSeed). Session 01 owns shared
contracts. Only touch if a field is genuinely absent — none expected. Import types via
`@purechess/shared`. Run `pnpm --filter @purechess/shared build` first so the import resolves.

**D10 — no raw WS strings / no enums needed here.** Engine client is UCI-over-postMessage, unrelated
to `WsEvent`. N/A.

## 3. Public API surface (final)

```ts
export class EngineTimeoutError extends Error {}
export class EngineCancelledError extends Error {}

export function getBestMove(fen: string, level: number, movetimeMs?: number): Promise<string>; // unchanged
export function analyze(fen: string, options?: EngineAnalysisOptions): Promise<EngineEval>;
export function getHint(fen: string, level: number): Promise<string>;       // UCI best move
export function cancel(): void;                                              // stop in-flight go
export function warmUp(): Promise<void>;                                     // resolves on uciok
```

## 4. Risks + mitigations

| Risk | Mitigation |
|---|---|
| Shared worker state leak (MultiPV/ELO/Skill carry over) | Emit all relevant setoptions before EVERY `go`; reset MultiPV=1 by setting it each call |
| Interleaved info/bestmove on shared worker (concurrent calls) | Serialize via `currentJob` promise-chain; one in-flight job |
| `bestmove` arrives after `cancel`/`stop` | Listener removed before resolve; late lines ignored; queue advances |
| Long movetime false-timeouts | Per-call timeout = max(movetimeMs+buffer, 10s) |
| Test env has no `Worker`/`WebAssembly` | Mock `global.Worker` in test; force `wasmSupported=false` path or stub WebAssembly; reset `workerPromise` between tests (module-level state) |
| Module-level singleton across tests | `vi.resetModules()` + dynamic `import()` per test, OR export a `__resetForTests()` test hook (prefer resetModules to keep prod surface clean) |
| Engine ELO range varies by build | Clamp UCI_Elo to [1320, 3190]; document |
| Non-determinism from seed default | Default seed 0 → deterministic first/best pick; never `Math.random` |
| Coverage gate | NOTE: 90/85 gate is `apps/api/src/chess/engine/` ONLY. This is web/Vitest — no enforced gate, but cover all new branches (parse cp vs mate, multipv, elo vs skill, cancel, timeout) |

## 5. Test plan — `apps/web/test/engine/stockfish-client.test.ts`

Mock strategy: define a `MockWorker` class assigned to `global.Worker` that records `postMessage`
calls and lets the test drive `onmessage`/dispatch `message` events. Use `vi.resetModules()` +
`await import(...)` per test so the module-level `workerPromise` is fresh. Stub `global.WebAssembly`
or rely on the `.js` (non-wasm) branch.

Cases:
1. **handshake/warmUp** — `warmUp()` posts `uci`, resolves after mock emits `uciok`.
2. **info parse: cp** — feed `info depth 12 score cp 34 multipv 1 pv e2e4 e7e5`; `analyze` resolves
   `{cp:34, depth:12, bestmove:'e2e4', pv:['e2e4','e7e5']}`.
3. **info parse: mate** — `score mate 3` → `{mate:3,...}`, no `cp`.
4. **multipv** — two info lines (multipv 1 & 2); analyze returns line 1; blunder path can reach line 2.
5. **getHint** — returns `pv[0]` UCI for side to move.
6. **ELO option emission** — `analyze(fen,{eloTarget:1500})` → asserts `postMessage` saw
   `setoption name UCI_LimitStrength value true` and `... UCI_Elo value 1500`.
7. **skill option emission** — level→skill bucket emitted; LimitStrength false.
8. **blunder determinism** — multiPv≥2 + blunderCp window + seed → same pick across two runs.
9. **think-time** — `movetimeMs:2500` → `go movetime 2500` posted.
10. **cancel** — start analyze, call `cancel()`, asserts `stop` posted and promise rejects with
    `EngineCancelledError`; late `bestmove` ignored.
11. **timeout type** — no bestmove emitted, advance timers (`vi.useFakeTimers`) → rejects with
    `EngineTimeoutError` (instanceof).
12. **getBestMove back-comat** — `getBestMove(fen, 4)` resolves UCI from `bestmove` line (existing
    behavior intact).
