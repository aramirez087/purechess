# Session 01 Implementation Plan — vs-computer Contracts Charter

**Scope:** Define single source-of-truth TypeScript contracts in `@purechess/shared` that
Sessions 02 (API), 03 (engine client), and 04 (web data layer) build against in parallel.
**Types + DTOs only. No endpoint logic, no UI logic, no service edits.**

---

## 0. Critical findings from repo recon (drive every decision below)

1. **Existing shared DTOs are plain TS `interface`s — there is NO class-validator anywhere
   in `packages/shared`.** `packages/shared/package.json` has zero runtime deps (only
   `typescript` devDep). `game.dto.ts`, `matchmaking.dto.ts`, `computer-game.dto.ts` are all
   decorator-free interfaces.
   - The session task text says "Use class-validator decorators matching existing DTO style."
     This is self-contradictory: the *existing style has no decorators*.
   - **Decision: keep plain interfaces.** Do NOT add class-validator to shared. Rationale:
     (a) matches actual existing style, (b) keeps the types-only package dependency-free and
     buildable with bare `tsc`, (c) runtime validation belongs in the API layer (Session 02
     can wrap these as NestJS validation classes if it chooses). Flagged in handoff.

2. **`ComputerGameStateDto` is constructed by a plain object literal in
   `apps/api/.../computer-games.service.ts` (`toStateDto`, line 47-59).** That file is OUT OF
   SCOPE this session (source edit forbidden; also `-r typecheck` must stay green).
   - **Therefore every field added to `ComputerGameStateDto` MUST be optional (`?:`)**, or
     `apps/api` typecheck breaks because `toStateDto` won't supply the new keys. Session 02
     populates them when it rewrites the service.
   - Same applies to `CreateComputerGameDto` new fields (already naturally optional).

3. **Reuse existing enums — all draw/abort reasons already exist** in
   `packages/shared/src/game-result.ts`:
   `GameTermination.{Stalemate, InsufficientMaterial, ThreefoldRepetition, FiftyMoveRule,
   DrawAgreement, Abandonment, Timeout, Resignation, Checkmate}`. **Do not invent new enums.**
   Abort maps to `GameTermination.Abandonment`. Draw-by-agreement = `DrawAgreement`; claim
   reasons = `ThreefoldRepetition` / `FiftyMoveRule` / `Stalemate`. `GameResult.Draw` for the
   result.

4. **Clock shape already serialized as numbers.** `apps/api/src/chess/engine/clock.ts`
   `serializeClock()` returns `{ whiteMs: number; blackMs: number; lastTickAt: number;
   incrementMs: number }` (internal `ClockSnapshot` uses `bigint`; the wire form is `number`).
   The new `clock` DTO field mirrors the serialized number form — never `bigint` (not
   JSON-safe).

5. **Empty-interface lint risk.** `AbortDto`/`RematchDto` are semantically bodyless. An empty
   `interface {}` can trip `@typescript-eslint/no-empty-object-type`. Mitigation: give them a
   single optional, meaningful field (see §2) so they are non-empty and still useful.

## 2. Exact target shapes

### `computer-game.dto.ts`

```ts
import type { GameResult, GameTermination } from '../game-result.js';

export interface CreateComputerGameDto {
  level: 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8;
  color: 'white' | 'black' | 'random';
  timeControlSeconds: number;          // confirmed present (required)
  incrementSeconds?: number;           // confirmed present
  eloTarget?: number;                  // NEW — UCI_Elo target (Session 03 strength mode)
  thinkTimeMs?: number;                // NEW — engine movetime per move
  styleBlunderCp?: number;             // NEW — ± centipawn window for human-like play
}

export interface ComputerMoveDto {
  move: string;                        // unchanged (UCI, or "resign")
}

// Serialized (number) clock — mirrors engine/clock.ts serializeClock(). Never bigint.
export interface ComputerClockDto {
  whiteMs: number;
  blackMs: number;
  lastTickAt: number;                  // epoch ms of last tick
  incrementMs?: number;
}

export interface ComputerGameStateDto {
  gameId: string;
  fen: string;
  pgn: string;
  status: string;                      // includes 'active' | 'completed' | 'aborted'
  computerColor: 'white' | 'black';
  computerLevel: number;
  lastComputerMove: string | null;
  result: string | null;
  resultReason: string | null;
  // --- NEW, all OPTIONAL so existing toStateDto literal still typechecks ---
  clock?: ComputerClockDto | null;     // NEW
  drawOffered?: boolean;               // NEW — a draw offer is pending
  drawOfferedBy?: 'white' | 'black' | null; // NEW — who offered (Session 02 has drawOfferedBy col)
  abortable?: boolean;                 // NEW — true iff zero player moves made
}

// ---- Request DTOs (plain interfaces, matching existing style) ----

export interface TakebackDto {
  plies: 1 | 2;                        // 1 = undo human move; 2 = undo human + bot reply
}

export interface RewindToPlyDto {
  ply: number;                         // truncate to this ply (>= 0)
}

export interface AbortDto {
  reason?: string;                     // optional; avoids empty-interface lint
}

export interface DrawActionDto {
  action: 'offer' | 'accept' | 'decline' | 'claim';
}

export interface RematchDto {
  swapColors?: boolean;                // optional; same settings, optionally swap side
}

export interface CreateFromFenDto extends CreateComputerGameDto {
  fen: string;                         // starting position; settings inherited
}
```

> Note: `GameResult`/`GameTermination` imported as `type` for downstream reference but the DTOs
> keep `result`/`resultReason` as `string | null` to match the current service literal (which
> writes raw strings). Do NOT tighten them to the enums this session — that would break the
> `toStateDto` literal in api. Document the enum mapping in the handoff for Session 02.

## 3. Design decisions & rationale

| Decision | Rationale |
|---|---|
| Plain interfaces, no class-validator | Matches actual existing style; shared stays dep-free + `tsc`-buildable. Validation is the API layer's job (Session 02). |
| All new `ComputerGameStateDto` fields optional | This session cannot edit `computer-games.service.ts`; `-r typecheck` must stay green. Optional fields let the existing literal compile unchanged. |
| `clock` uses `number` (serialized), not `bigint` | `bigint` is not JSON-serializable; mirrors `serializeClock()` wire shape exactly. |
| Reuse `GameTermination` (Abandonment for abort, existing draw reasons) | Operator rule + task §4: do not invent enums. |
| `result`/`resultReason` stay `string \| null` | Current service writes raw strings into the literal; tightening to enums breaks api typecheck. Enum mapping documented for Session 02 instead. |
| `AbortDto.reason?` / `RematchDto.swapColors?` non-empty | Avoid empty-interface lint; both fields are plausibly useful. |
| `CreateFromFenDto extends CreateComputerGameDto` | DRY — inherits level/color/time-control/strength knobs; only adds `fen`. |
| `drawOfferedBy` added alongside `drawOffered` | Session 02 spec mentions a `drawOfferedBy` schema column; expose both so web can render "opponent offered a draw". |
| No override JSON written (pending §6 check) | No downstream `produces` already satisfied; nothing must move. |

## 4. Risks & mitigations

| Risk | Mitigation |
|---|---|
| Adding required fields to `ComputerGameStateDto` breaks `apps/api` typecheck | **All new state fields optional** — primary mitigation, verified by `pnpm -r typecheck`. |
| `bigint` leaking into a DTO → JSON serialize crash downstream | Clock DTO uses `number`; documented. |
| Empty-interface lint failure on Abort/Rematch | Each carries one optional field. |
| `@typescript-eslint/no-explicit-any` or import-style lint | Use `import type` with explicit `.js` ext (ESM rule); no `any`. |
| Task text vs reality conflict (class-validator) | Decision documented in plan + handoff; reviewer can object before Session 02. |
| Shared not rebuilt → apps typecheck against stale `dist` | Run `pnpm --filter @purechess/shared build` BEFORE `-r typecheck` (order enforced in §5). |
| Override-file expectation by epic tooling | Re-check `.epic-status.json`/dag in §6; write file only if tooling keys on it. |
