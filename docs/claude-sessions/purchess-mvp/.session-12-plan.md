## Key Design Decisions

### 1. Component API
```ts
interface ChessboardProps {
  position: string           // FEN string ŌĆö source of truth
  orientation: 'white' | 'black'
  legalMoves: Move[]         // from server; board uses chess.js only for UX preview
  onMove: (from: Square, to: Square, promotion?: PromotionPiece) => void
  lastMove?: { from: Square; to: Square }
  checkSquare?: Square
  premove?: { from: Square; to: Square } | null
  settings?: Partial<BoardSettings>
  readOnly?: boolean         // review mode (session 18): no input, no premoves
}
```

**Rationale**: Fully presentational. Parent owns server state. Board owns drag/click/premove UX state internally. `legalMoves` from server prevents client from diverging; chess.js used only for instant legal-move preview highlights (before server round-trip).

### 2. Drag implementation ŌĆö pointer events only
Use `onPointerDown` on `<Piece>`, capture pointer with `element.setPointerCapture(e.pointerId)`, listen `onPointerMove`/`onPointerUp` on the board container. No native HTML5 drag. `touch-action: none` on the board grid.

A drag ghost (absolutely-positioned clone following pointer via CSS `transform`) is rendered outside the 8├Ś8 grid ŌĆö it's a sibling div, `pointer-events: none`, `z-index: 50`. This avoids layout thrash during drag.

Source square dims (opacity 0.3) during drag.

**Rationale**: Pointer events unify mouse+touch. Native drag is broken on Linux/Android.

### 3. Click-to-move state machine
Internal state in `useBoardInput`:
- `idle` ŌåÆ `onPointerDown` on own piece ŌåÆ `selected(from)` ŌåÆ show legal dots
- `selected(from)` ŌåÆ `onPointerDown` on legal dest ŌåÆ emit `onMove(from, dest)`
- `selected(from)` ŌåÆ `onPointerDown` on own piece ŌåÆ `selected(newFrom)` (re-select)
- `selected(from)` ŌåÆ `onPointerDown` on board container (not piece/legal-sq) ŌåÆ `idle`
- `selected(from)` ŌåÆ drag threshold exceeded ŌåÆ enters drag mode, deselects

Click and drag share the same `onPointerDown` entry point; drag is detected if pointer moves >4px before `onPointerUp`.

### 4. Premove
Only active when `readOnly=false` and it is the opponent's turn (detected: color to move in FEN ŌēĀ player orientation). Premove stored in Chessboard local state (not parent). Ghost piece shown at premove dest. On receiving new `position` prop:
1. Re-validate premove against new FEN with chess.js
2. If legal: call `onMove` immediately, clear premove
3. If illegal: clear silently

One premove at a time ŌĆö new premove replaces old.

### 5. Piece SVGs
Full set: K, Q, R, B, N, P ├Ś 2 colors = 12 components in `piece-svgs.tsx`. Each component accepts `className` prop. Fill uses `currentColor`; stroke contrasts (hardcoded as a fraction lighter/darker, not currentColor). White pieces: light body, dark stroke. Black pieces: dark body, light stroke.

Both piece styles ship in one file ŌĆö no network calls, no sprite sheets.

### 6. Board CSS variables
Added to `globals.css` under `:root`:
```css
--board-sq-light: 30 20% 85%;   /* warm off-white */
--board-sq-dark:  30 25% 48%;   /* warm brown */
--board-sq-size: 64px;          /* overridden by ResizeObserver */
--board-highlight-last: 60 100% 50%;
--board-highlight-selected: 200 100% 60%;
--board-highlight-legal: 0 0% 0%;
--board-highlight-check: 0 72% 51%;
--board-highlight-premove: 210 80% 60%;
```

`[data-theme="dark"]` gets slightly different board colors (cooler tones). Board colors intentionally not tied to `--background`/`--card` ŌĆö chess boards have their own palette.

### 7. Animations
`piece.tsx` applies a `data-animating` attribute + CSS transition `transform 200ms ease-out` when a move arrives (position prop change detected). Castle: both rook and king get animations simultaneously (React concurrent renders in same frame). En passant captured pawn: fade out via opacity transition.

`prefers-reduced-motion`: detected once via `window.matchMedia`. When true, no transition classes added. Pure position update.

### 8. Sound (Web Audio API)
`sound.ts` creates an `AudioContext` lazily (on first user gesture). Each sound type is a short synthesized tone:
- `move`: square wave, 880Hz, 30ms decay
- `capture`: lower square wave, 660Hz, 50ms decay  
- `check`: two simultaneous oscillators (880 + 1320Hz), 80ms
- `mate`: descending 4-tone sequence
- `game-start`: rising 3-tone arpeggio

All are generated tones ŌĆö zero network requests. Volume controlled via `GainNode`.

### 9. ResizeObserver
`useBoardResize(ref)` observes the board container, computes `Math.floor(container.width / 8)` as square size, writes to `--board-sq-size` CSS custom property on the element. Squares use `width: var(--board-sq-size); height: var(--board-sq-size)`. Board max-width: 720px on desktop, full width on mobile (via Tailwind `max-w-[720px] w-full`).

### 10. Test setup
Vitest + jsdom + @testing-library/react. `vitest.config.ts` extends tsconfig paths. Tests in `apps/web/test/board/`.

Pure tests (`position.test.ts`, `premove.test.ts`): no DOM, just function calls.
DOM tests (`board.test.tsx`): render `<Chessboard>`, simulate events via userEvent.

## Risks and Mitigations

| Risk | Mitigation |
|---|---|
| SVG piece set is large ŌĆö bundle bloat | Tree-shaking via named exports; each piece is a small functional component. Measure gzip at end. |
| Drag ghost causes paint jank | `will-change: transform` on ghost div; no layout properties during drag (only `transform`). |
| Pointer capture lost on fast swipes | Set capture on `pointerdown`, release on `pointerup` AND `pointercancel`. |
| chess.js FEN parsing edge cases (960, etc.) | Only used for legal-move preview on standard chess positions ŌĆö server is authoritative. Document this. |
| Audio context blocked by browser autoplay policy | Lazy init: create `AudioContext` only inside a user gesture handler (`pointerdown`). |
| ResizeObserver fires during animation | Debounce 16ms (one frame) to prevent thrash. |
| Promotion auto-triggered on pawn to last rank | Show inline picker, block `onMove` until user picks piece. |
| Test runner not configured | Add vitest to devDependencies; add `"test"` script to `apps/web/package.json`. |
| `chess.js` Move type may differ from our Square type | Define local `Square` type as `'a1'`ŌĆ”`'h8'` string literal union; cast carefully. |
