## Key Design Decisions

### 1. Component API
```ts
interface ChessboardProps {
  position: string                  // FEN — source of truth
  orientation?: 'white' | 'black'   // default 'white'
  legalMoves?: Move[]               // from server; omit → read-only
  onMove?: (move: MoveIntent) => void
  lastMove?: { from: Square; to: Square }
  checkSquare?: Square
  className?: string
  readOnly?: boolean                // session 18 review mode
}
```
Fully presentational. Parent owns game state. Board owns drag/click/premove UX internally. `chess.js` is used only for instant legal-move preview highlights (before server confirmation). Server `legalMoves` is the authoritative source.

### 2. Pointer events for drag (not HTML5 drag API)
`onPointerDown` on `<Piece>` → `element.setPointerCapture(e.pointerId)` → `onPointerMove`/`onPointerUp` on board container. `touch-action: none` on the grid. Drag ghost is an absolutely-positioned sibling div (`pointer-events: none`, `z-index: 50`) that follows pointer via `transform: translate(x,y)` — no layout thrash.

Source square dims (opacity 0.3) during drag. Snap-back if dropped on illegal square: brief CSS transition back to origin, then remove ghost.

### 3. Click-to-move state machine (inside `use-click-move.ts`)
```
idle
  → pointerdown on own piece → selected(from)
selected(from)
  → pointerdown on legal dest → emit onMove, → idle
  → pointerdown on own piece (different) → selected(newFrom)
  → pointerdown on empty/opponent/illegal → idle
  → pointer moves >4px before pointerup → enter drag mode
```
Click and drag share the same `pointerdown` entry point in `use-drag.ts`; drag detection threshold is 4px movement.

### 4. Premove (board-local, transparent to parent)
Active only when `readOnly=false` and FEN turn ≠ player orientation. Stored in `Chessboard` local state (not exported). Ghost piece shown at destination. On new `position` prop (opponent moved):
1. Re-validate premove against new FEN with chess.js
2. Legal → emit `onMove` immediately, clear premove
3. Illegal → clear silently

One premove at a time — new replaces old.

### 5. Piece SVGs
12 inline SVG components in `piece-svgs.tsx`. Fill: `currentColor`. White pieces: light body + dark stroke hardcoded. Black pieces: dark body + light stroke hardcoded. ViewBox: `0 0 45 45` (standard chess piece proportions). `getPieceSvg(type: PieceType, color: Color): React.FC` dispatcher enables memoized lookup.

### 6. Board CSS variables (added to `globals.css`)
```css
:root {
  --board-sq-light: 30 20% 85%;
  --board-sq-dark:  30 25% 48%;
  --board-sq-size:  64px;            /* overwritten by ResizeObserver */
  --board-highlight-last:     60 100% 50% / 0.4;
  --board-highlight-selected: 200 100% 60% / 0.6;
  --board-highlight-check:    0 72% 51% / 0.55;
  --board-highlight-premove:  210 80% 60% / 0.35;
}
[data-theme="dark"] {
  --board-sq-light: 220 15% 55%;
  --board-sq-dark:  220 25% 32%;
}
@keyframes check-pulse {
  0%, 100% { background-color: hsl(0 72% 51% / 0.55); }
  50%       { background-color: hsl(0 72% 51% / 0.25); }
}
/* animation-iteration-count: 1; animation-fill-mode: forwards */
```
Legal-move dots: small `::before` pseudo-element in `square.tsx` (dark semi-transparent circle for non-capture, ring for capture) — no extra DOM nodes.

### 7. Animations — CSS transitions only
On FEN change, `use-animations.ts` computes `{from, to}` pairs. Moving piece gets `data-animating="true"` for one JS frame (via `requestAnimationFrame`), then class removed. CSS: `[data-animating] { transition: transform 200ms ease-out; }`. Rook and king both animated in castle (same frame). En passant: captured pawn gets `data-fading` → `opacity: 0; transition: opacity 200ms`. `prefers-reduced-motion` → no data-animating/data-fading attributes.

### 8. Sound — Web Audio API, no files
`SoundEngine` creates `AudioContext` lazily on first user gesture (`pointerdown`). Each sound: short oscillator + `GainNode` envelope. Stored as singleton in module scope. `soundEngine.play('move' | 'capture' | 'check' | 'mate' | 'game-start')`. Volume knob via `GainNode.gain`.

### 9. ResizeObserver
`useBoardResize(containerRef)` — `Math.floor(containerWidth / 8)` px per square → written as `element.style.setProperty('--board-sq-size', px + 'px')` directly on the container element (avoids React re-render). Debounced 16ms. Board layout: `max-w-[720px] w-full mx-auto`.

### 10. Keyboard navigation
`tabIndex={0}` on board container. Internal `focusSquare: {file: 0-7, rank: 0-7}` state in `use-keyboard.ts`. ArrowUp/Down/Left/Right move cursor. Enter: if idle → select piece; if selected + focus on legal dest → emit `onMove`. Esc: deselect. Focus ring: `focus-visible:ring-2 ring-ring` on board container (not individual squares — too noisy for AT).

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| chess.js v1 API differences | Use `new Chess(fen)`, `.moves({ square, verbose: true })`, `.inCheck()`. Test with known FENs before using in components. |
| SVG bundle size | Named exports enable tree-shaking. 12 components × ~200 bytes minified ≈ 2.4KB raw. Target: pieces < 5KB gz. |
| Pointer capture lost on fast swipes | Listen `pointercancel` on container, treat same as `pointerup` — reset drag state. |
| Audio blocked by browser autoplay | Lazy AudioContext creation inside `pointerdown` handler — always a user gesture. |
| ResizeObserver fires during drag animation | 16ms debounce absorbs single-frame bursts. |
| Promotion: pawn reaches 8th from non-standard FEN | `isPromotion(from, to, fen)` checks rank + piece type. Picker always shown; never auto-promotes. |
| No test runner configured | Step 1 in order of operations — vitest added before any component code. |
| jsdom missing `ResizeObserver` | Mock in `vitest.setup.ts`: `global.ResizeObserver = class { observe(){} unobserve(){} disconnect(){} }` |
| chess.js `Move` type vs shared `Square` type | Both use `'a1'`…`'h8'` string literals. Cast chess.js move squares via `as Square`. |
