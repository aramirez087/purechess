## Key Design Decisions

### 1. CtaButton wraps Link via asChild
CTAs are navigation → render as `<Link>` with button styling. Use shadcn `Button` with `asChild` prop + `<Link>` as child. Mapping:
- `primary` → shadcn `default` + `size="lg"` + `w-full sm:w-auto`
- `secondary` → shadcn `outline` + `size="lg"`
- `tertiary` → shadcn `ghost` + `size="sm"` + muted styling

### 2. "Analyze a game" CTA — disabled stub
Disabled button (not a link — disabled links are a11y antipattern) wrapped in Tooltip showing "Coming soon". Uses Radix `TooltipProvider` + `Tooltip` from `@/components/ui/tooltip`. Satisfies "three CTAs" requirement without lying about functionality.

### 3. "Play now" route — query param, no localStorage
`/play?mode=casual` carries the mode in the URL. The `/play` page (s13) reads `searchParams.mode`. No client JS needed, no localStorage, no `'use client'` on the hero. Keeps hero fully server-rendered.

### 4. Hero entrance animation
Use `tailwindcss-animate` plugin — shadcn installs it. Class `animate-in fade-in duration-500` on the hero container. If plugin absent from tailwind config, add a single `@keyframes fadeIn` + utility class in `globals.css` instead of adding a dep.

### 5. Page structure
```
<AppShell variant="default">   ← top bar
  <div className="flex flex-col flex-1">
    <Hero />                    ← h1 wordmark, tagline, 3 CTAs
    <TrustStrip />              ← 4 statements
    <Footer />                  ← copyright + links, mt-auto to push down
  </div>
</AppShell>
```
AppShell wraps children in `<main id="main-content">`. Footer inside main is acceptable for MVP — semantically it is the page footer, and the `<footer>` element is used regardless of parent context.

### 6. SEO helper — `lib/seo.ts`
```ts
export function buildMetadata(overrides?: Partial<Metadata>): Metadata
```
Returns base site metadata merged with overrides. Base includes title template, description, OG defaults. Homepage exports `metadata = buildMetadata()` with page-specific title/description. `layout.tsx` keeps its own `metadata` export; `page.tsx` overrides with the homepage copy.

### 7. OG image
Static PNG at `public/og-image.png` (placeholder). URL constructed as `${process.env.NEXT_PUBLIC_BASE_URL}/og-image.png`. Defaults to `https://purchess.com` if env var missing. Document `NEXT_PUBLIC_BASE_URL` in `.env.example`.

### 8. Test framework — vitest
Add to `devDependencies`:
- `vitest`, `jsdom`, `@testing-library/react`, `@testing-library/jest-dom`, `@testing-library/user-event`, `@vitejs/plugin-react`

`vitest.config.ts` uses jsdom environment. `src/test-setup.ts` imports `@testing-library/jest-dom`.

Tests assert:

- "Analyze" button is disabled

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| `tailwindcss-animate` absent from tailwind config | Check `tailwind.config.ts`; if plugin missing, add `@keyframes fadeIn` to `globals.css` and a `.fade-in` utility — no new dep |
| vitest + Next.js module resolution conflicts | Configure `resolve.alias` for `@/` in vitest config; mock `next/navigation` and `next/link` in test setup |
| `NEXT_PUBLIC_BASE_URL` unset → OG URL relative | Default to `https://purchess.com` constant in `seo.ts` |
| Footer inside `<main>` semantically debatable | Use `<footer>` element; for MVP this is fine — note in handoff for future refactor |
| `page.tsx` exports `metadata` — conflicts with AppRouter conventions? | No conflict; RSC page files can export both `metadata` and a default component |
