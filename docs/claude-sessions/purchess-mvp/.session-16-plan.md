### Step 2 — `lib/seo.ts`

```ts
import type { Metadata } from 'next';

export const SITE_URL = process.env.NEXT_PUBLIC_SITE_URL ?? 'https://purchess.com';

type BuildMetadataInput = {
  title: string;
  description: string;
  canonical: string;
  ogImage?: string;
};

export function buildMetadata({
  title,
  description,
  canonical,
  ogImage = '/og-image.png',
}: BuildMetadataInput): Metadata {
  return {
    title,
    description,
    alternates: { canonical },
    openGraph: {
      title,
      description,
      type: 'website',
      url: canonical,
      images: [{ url: ogImage }],
    },
    twitter: {
      card: 'summary_large_image',
      title,
      description,
      images: [ogImage],
    },
  };
}
```

### Step 3 — `cta-button.tsx`

Wraps shadcn `Button` with `asChild` + Next.js `Link`. Variant mapping:

- `primary` → shadcn `default` variant, `size="lg"`, full-width on mobile
- `secondary` → shadcn `outline` variant, `size="lg"`
- `tertiary` → shadcn `ghost` variant, `size="sm"`

Props: `variant`, `href?: string`, `disabled?: boolean`, `className?`, `children`.

When `href` present and not disabled → render `<Button asChild><Link href={href}>`. When `disabled` → render `<Button disabled>` (no Link). This avoids a broken `<a>` with no href.

Disabled "Analyze" CTA uses TooltipProvider + Tooltip wrapping a `<span tabIndex={0}>` around the disabled button (disabled buttons block pointer events; span wrapper keeps keyboard-accessible tooltip trigger).

**Mode param decision**: Use query param `?mode=casual` only. No localStorage inline script. Simpler, fully testable, zero client JS on homepage. The `/play` page (session 13) must read this param when built. Document in handoff.

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| `AppShell variant="minimal"` | Wordmark in h1 IS the page identity; top nav would duplicate it and add distraction |
| Query param `?mode=casual` not localStorage | Zero client JS on homepage; fully testable; `/play` page reads param when built |
| `CtaButton` wraps shadcn Button + asChild | Reuses existing variant/focus ring system; single source of truth |
| Tailwind keyframe, no plugin | `tailwindcss-animate` not installed; single animation, zero new deps |
| Disabled Tooltip via `<span tabIndex>` wrapper | Disabled buttons block pointer events; span wrapper keeps keyboard-accessible |
| `new Date().getFullYear()` in footer | Server-rendered; no hydration mismatch; accurate without client JS |
| OG image: `/og-image.png` placeholder | Static asset not in scope this session; documented for downstream |
| `NEXT_PUBLIC_SITE_URL` env var | No hardcoded domain; canonical and OG URLs correct per environment |

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| No Jest deps exist | Plan adds all required devDeps; `next/jest` transformer handles TSX/Next internals |
| `setupFilesAfterFramework` typo risk | Correct key is `setupFilesAfterFramework` — double-check during impl |
| AppShell minimal: skip-link target | `id="main-content"` already on `<main>` in AppShell; skip-link in layout.tsx still works |
| Tooltip needs TooltipProvider high up | Use per-hero `<TooltipProvider>`; single instance, no need to hoist to providers.tsx |
| Dark mode trust-strip dividers | `border-border` is theme-aware; tested in both themes manually |
| Lighthouse CI | No headless browser in session env; document as manual verification; RTL tests cover functional correctness |
| `jest.config.ts` — `setupFilesAfterFramework` vs `setupFilesAfterEach` | Correct key is `setupFilesAfterFramework` — verify against jest docs |
