## Key Design Decisions

### 1. `CtaButton` wraps `Button asChild` + `Link`

The shadcn `Button` supports `asChild` (Radix Slot). `CtaButton` passes `asChild` to render a `<Link>` as the button element — no wrapper div, correct semantic HTML, focus ring from shadcn preserved.

Variant mapping:
- `primary` → `variant="default"` size `lg` + `w-full sm:w-auto`
- `secondary` → `variant="outline"` size `lg`
- `tertiary` → `variant="ghost"` size `sm` + `text-muted-foreground`

### 2. `AnalyzeCta` client island

Shadcn `Tooltip` requires `onMouseEnter` / pointer events — needs `'use client'`. Isolate this to a single small component so the rest of the hero stays a server component (zero client JS for static parts).

The "Play now" and "Create account" CTAs are plain server-rendered `<Link>` elements — no hydration cost.

### 3. `mode=casual` via query param, not localStorage

The session spec says `→ /play with mode=casual query`. A plain `href="/play?mode=casual"` is server-renderable and doesn't need inline scripts. The `/play` page (session 13) can read `searchParams.mode`. This is simpler and avoids client JS.

### 4. SEO helper `buildMetadata`

Returns a `Metadata` object merged with per-page overrides. Canonical URL constructed from `NEXT_PUBLIC_BASE_URL` env var (falls back to `https://purchess.app` if unset — safe for build time).

```ts
export function buildMetadata(overrides?: Partial<Metadata>): Metadata
```

Homepage uses: title, description, canonical, OG (title, description, type=website, image), Twitter card summary_large_image.

OG image path: `/og-image.png` (placeholder — a future session can generate a real OG image; for MVP the file can be a simple 1200×630 wordmark PNG placed in `public/`).

### 5. Fade-in animation

`@keyframes fade-in` in `globals.css` (or Tailwind `animate-` utility if already present). Use `animate-[fadeIn_0.4s_ease-out]` on the hero wrapper. No JS, no scroll-jacking. Check if Tailwind config already has a `fadeIn` keyframe from session 03 — if not, add to `globals.css` `@layer utilities`.

### 6. No test framework

`apps/web/package.json` has no jest/vitest. The session requires unit tests (renders wordmark, CTAs, trust statements, footer; checks links). Two options:
- **Option A (preferred)**: Install `vitest` + `@testing-library/react` + `jsdom` as devDependencies and wire up `vitest.config.ts`. This is minimal setup (~4 files).
- **Option B**: Note gap in handoff, defer tests to session 25 (e2e/QA). Risk: session is not "done" per DoD.

**Decision**: Go with Option A — install vitest. The session explicitly requires tests and Lighthouse targets (a11y ≥ 95). Tests are the only verifiable part without a browser.

Lighthouse scores cannot be measured in this headless environment — note this in handoff (same pattern as session 03 for a11y).

### 7. Dark mode

All color classes use design tokens (`bg-background`, `text-foreground`, `text-muted-foreground`, `border-border`). `dark:` variants work automatically per session 03 tailwind config. No extra work needed.

### 8. Homepage wraps in `AppShell variant="default"`

The homepage should render the full top bar (logo, nav, user menu). `app/page.tsx` wraps content in `<AppShell>`. The hero section sits below the top bar, centered.

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| No OG image file exists | Use `/og-image.png` path in metadata; create a 1×1 placeholder PNG in `public/` so build doesn't 404. Note in handoff that a real image is needed. |
| Vitest setup may conflict with Next.js | Use `@vitejs/plugin-react` + `jsdom` environment. Exclude `apps/web/src/app/` (Next.js server components can't run in jsdom). Test only the component files in `components/home/`. |
| `AppShell` passes `user={null}` hardcoded | Pre-existing (noted in session 03 handoff). Auth wiring is a future session. |
| `/play`, `/register`, `/games` routes don't exist yet | Links are valid `<Link>` elements — they navigate. 404 pages handled by `not-found.tsx`. No risk to this session. |
| `animate-[fadeIn_...]` syntax requires Tailwind JIT | Next.js 14 with Tailwind 3.4 uses JIT by default. Add keyframe to `tailwind.config.ts` `theme.extend.keyframes`. |
