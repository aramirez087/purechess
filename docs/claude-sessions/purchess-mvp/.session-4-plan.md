### `apps/api/src/auth/sessions.service.ts`
Session lifecycle over Prisma `Session` table:
- `createSession(userId, ipAddress?, userAgent?): Promise<{ token: string; expiresAt: Date }>`
  - 32 random bytes → base64url → cookie token
  - HMAC-SHA256(SESSION_SECRET, rawToken) → hex → stored as `session.id`
  - No schema migration needed; override `@default(cuid())` with explicit id
  - expiresAt = now + 30 days
- `lookupSession(token: string): Promise<{ session: Session; user: User } | null>`
  - Recompute HMAC → `findUnique({ where: { id: hmac }, include: { user: true } })`
  - Return null if not found, expired, or user disabled
  - Sliding window: if expiresAt < now+7d, extend to now+30d
- `revokeSession(token: string): Promise<void>` — delete by HMAC id
- `revokeAllUserSessions(userId: string): Promise<void>` — deleteMany

### `apps/api/src/auth/auth.service.ts` (full replace)
Methods:
- `register(dto, ip?, ua?)` — find existing user by email/username (throw 409 if found), hashPassword, create User, create Rating rows for all 3 categories, createSession, return AuthResponse
- `login(dto, ip?, ua?)` — find user by email OR username (citext handles case), verifyPassword (throw 401 if wrong), createSession, return AuthResponse
- `logout(token?)` — revokeSession (no-op if undefined)
- `requestPasswordReset(email)` — always returns void; if user found: 32 random bytes → hex token, SHA-256(token) → tokenHash, store PasswordResetToken (1h expiry), log URL in dev

Helper: `setCookie(res, token, expiresAt)` — sets `purchess_session` with httpOnly, sameSite:Lax, secure in prod, maxAge

## Key Design Decisions

### Session token storage (no schema migration needed)
- Cookie: `base64url(crypto.randomBytes(32))` (43 chars)
- DB id: `createHmac('sha256', SESSION_SECRET).update(rawToken).digest('hex')` (64 hex chars)
- Session.id `@default(cuid())` is overridden by passing explicit id to `prisma.session.create`
- Lookup: recompute HMAC → `findUnique({ where: { id: hmac } })`
- Rationale: No schema migration, timing-safe, forward-compatible

### Global prefix `/api`
- `app.setGlobalPrefix('api')` in main.ts is the cleanest approach
- All controllers keep short names: `@Controller('auth')`, `@Controller('games')`, etc.
- This is a controlled out-of-touches edit justified by platform-level wiring

### SESSION_SECRET replaces JWT_SECRET
- No JWTs are used in this auth design
- JWT_SECRET was scaffolded in session 01 as a placeholder
- Rename + raise min length to 32 chars
- Note in handoff that downstream sessions should not add JWT auth without a dedicated session

### Argon2 for passwords only
- `bcryptjs` stays in package.json (seed.ts already uses it for seeding test passwords)
- New `PasswordService` uses `argon2` (argon2id variant, default params)
- Cost params: timeCost=3, memoryCost=65536 (64MB), parallelism=1

### PasswordResetToken storage
- `PasswordResetToken.tokenHash` column already exists in schema
- Token: `crypto.randomBytes(32).toString('hex')` (64 hex chars) → sent in URL
- Stored: `createHash('sha256').update(token).digest('hex')` in tokenHash (no HMAC needed — 64-byte entropy)
- Expiry: 1 hour; one-time via `usedAt`

### Apple OAuth graceful degradation
- If `OAUTH_APPLE_TEAM_ID` is empty at startup: log warning, skip AppleOAuthStrategy registration
- Apple strategy is conditional via factory in auth.module.ts

### Rate limiting
- ThrottlerModule uses in-memory store (no Redis needed for auth throttling)
- Route-level overrides via `@Throttle({ default: { limit: N, ttl: Ms } })`
- Login + register: 5/min
- password-reset/request: 1/5min

## Risks & Mitigations

| Risk | Mitigation |
|---|---|
| argon2 native bindings require build tools | Document in handoff; verify pnpm install works; fallback comment in PasswordService |
| passport-apple requires private key PEM | Graceful skip when env vars blank; dev just uses Google |
| `app.setGlobalPrefix('api')` breaks existing `/health` or other routes | Check if other controllers exist; add `{ exclude: ['health'] }` if needed |
| bcryptjs in seed.ts vs argon2 in auth — test seed users won't pass verifyPassword | seed.ts uses raw bcrypt; auth login uses argon2.verify which rejects bcrypt hashes → seed users can't login via password. Mitigation: update seed.ts to hash with argon2 OR note in handoff that seed users need password reset. Preferred: update seed.ts hash to argon2. |
| Session HMAC id length (64 hex) exceeds Prisma String default | String in Prisma/PostgreSQL has no length limit by default — no issue |
| ThrottlerGuard on OAuth callback endpoints breaks redirect flow | Don't apply ThrottlerGuard to OAuth callback routes |
| package.json is outside touches scope | Listed as controlled out-of-touches; justified same as session 02 |
