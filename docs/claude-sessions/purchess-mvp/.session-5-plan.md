## Key Design Decisions

### 1. `GameResultReason` vs `GameTermination`
Session spec names `GameResultReason`. Shared already exports `GameTermination` with identical values. **Decision**: Add `GameResultReason` to `packages/shared/src/chess.ts` as a type alias for `GameTermination` (`export type GameResultReason = GameTermination`). Keeps wire compatibility with session 08 which will import both. No new enum needed.

### 2. EngineState — serializable form in shared vs api-internal form
Session spec has two forms:
- **API-internal** `EngineState`: contains `Chess` instance (not serializable)
- **Wire/shared** `SerializableEngineState`: plain JSON, stored in Redis

Approach: define `SerializableEngineState` in `packages/shared/src/chess.ts`. Define api-internal `EngineState` (with `Chess` instance) only in `apps/api/src/chess/engine/game-state.ts`. The engine converts to/from serializable form for Redis.

### 3. Pure functions vs NestJS service
Session says "EngineService (or pure functions)". **Decision**: pure functions in engine files + thin `EngineService` NestJS injectable that wraps them. `GameService` (session 08) injects `EngineService`. No circular deps since `ChessEngineModule` has no dependency on `GamesModule`.

### 4. `chess.js` cloning strategy
`Chess` is mutable. Pure-function discipline requires clone before every mutation. **Strategy**: use `new Chess(existingChess.fen())` to rehydrate — cheap since FEN captures full position state. This loses move history in the chess.js instance but we track moves in `EngineState.moves` ourselves, so that's fine. Threefold detection uses `fenHistory` array, not chess.js internals.

### 5. BigInt for clock math
Session requires BigInt internally. **Decision**: `clock.ts` accepts/returns `number` on public API (ms, safe integer range covers 24h games), uses `BigInt` only for internal arithmetic to prevent float drift. Serializes as `number` for wire/DB.

### 6. 50-move rule halfmove counter
chess.js FEN 5th field is halfmove clock. Parse from `chess.fen()` after each move. Store `halfMoveClock` in `EngineState.clock` (or on `EngineState` directly).

### 7. Jest setup
No jest config yet. Add jest config to `apps/api/package.json` (standard NestJS pattern with ts-jest). Tests live in `apps/api/test/chess/`.

## Exact Order of Operations

1. **Install chess.js + jest deps**
   ```
   pnpm --filter @purchess/api add chess.js
   pnpm --filter @purchess/api add -D jest ts-jest @types/jest
   ```

2. **Update `packages/shared/src/chess.ts`**
   - Add `GameResultReason` alias
   - Add `EngineMove` / `WireMove` interface
   - Add `SerializableEngineState` interface (JSON-safe)

3. **Create `apps/api/src/chess/engine/types.ts`**
   - `EngineMove` (internal, same shape as wire but with typed fields)
   - `EngineState` (api-internal, has `position: Chess`)
   - `CreateGameOptions` param type

4. **Create `apps/api/src/chess/engine/fen-utils.ts`**
   - `startingFen(): string`
   - `parseFen(fen: string): { turn: Color, halfMoveClock: number, fullMoveNumber: number }`
   - `toFen(state: EngineState): string`
   - `clonePosition(chess: Chess): Chess` — internal helper via `new Chess(chess.fen())`
   - `halfMoveClockFromFen(fen: string): number`

5. **Create `apps/api/src/chess/engine/clock.ts`**
   - `tickClock(state: EngineState, nowMs: number): EngineState`
   - `applyIncrement(state: EngineState, side: Color): EngineState`
   - `isTimeout(state: EngineState, nowMs: number): boolean`
   - `serializeClock(state: EngineState): { whiteMs: number; blackMs: number; lastTickAt: number; incrementMs: number }`
   - BigInt used internally, number on API surface

6. **Create `apps/api/src/chess/engine/result-detector.ts`**
   - `detectResult(state: EngineState, nowMs: number): { result: GameResult; reason: GameResultReason } | null`
   - Checks in order: timeout → checkmate → stalemate → insufficient material → threefold → fifty-move
   - Uses chess.js: `isCheckmate()`, `isStalemate()`, `isInsufficientMaterial()`, `isThreefoldRepetition()`
   - Threefold: count occurrences of current FEN (position part only, strip clocks) in `fenHistory`
   - 50-move: parse halfmove clock from current FEN ≥ 100

7. **Create `apps/api/src/chess/engine/move-validator.ts`**
   - `validateMove(state: EngineState, move: { uci?: string; san?: string }, nowMs: number): { ok: true; newState: EngineState } | { ok: false; reason: string }`
   - Steps: check status=active, tickClock → check timeout, parse move, attempt on cloned Chess instance, reject if null result, build new EngineState
   - UCI format: `e2e4`, `e7e8q` (promotion suffix required for pawn moves to 8th/1st rank)
   - Promotion: if pawn to last rank and no promotion piece → `{ ok: false, reason: 'PROMOTION_REQUIRED' }`

8. **Create `apps/api/src/chess/engine/game-state.ts`**
   - `createGame(opts: CreateGameOptions): EngineState`
   - `applyMove(state: EngineState, move: MoveInput, nowMs: number): EngineState` — calls validateMove internally, throws on invalid
   - `unmakeMove(state: EngineState): EngineState` — pop last move, rehydrate Chess from `fenHistory[n-1]`
   - `serializeState(state: EngineState): SerializableEngineState` — for Redis
   - `deserializeState(raw: SerializableEngineState): EngineState` — from Redis

9. **Create `apps/api/src/chess/engine/pgn-builder.ts`**
   - `buildPgn(moves: EngineMove[], headers: PgnHeaders): string`
   - Required headers: `Event`, `Site`, `Date`, `White`, `Black`, `Result`, `TimeControl`
   - Optional: `WhiteElo`, `BlackElo`, `ECO`
   - Format: standard PGN with move numbers, SAN, result token

10. **Create `apps/api/src/chess/engine/index.ts`** — barrel re-export

11. **Create `apps/api/src/chess/engine.service.ts`**

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| chess.js ESM-only in v1+ vs CJS requirement of NestJS | Use `chess.js ^1.1.0`; if ESM issue, use `"moduleNameMapper"` in jest config or switch to dynamic import with interop. Fall back to `chess.js@0.13.x` (CJS) if needed. |
| `noUnusedLocals/noUnusedParameters` in tsconfig.base.json | api tsconfig already overrides both to `false` — no issue |
| `GameTermination` vs `GameResultReason` naming collision | Use `export type GameResultReason = GameTermination` alias; both names valid downstream |
| chess.js `isThreefoldRepetition()` tracks its own history | We maintain `fenHistory` ourselves; compare stripped FENs (first 4 fields, drop halfmove+fullmove) to handle clock-agnostic repetition detection. Use chess.js `isThreefoldRepetition()` as a cross-check in tests but not as primary detection. |
| `strict: true` + BigInt vs number interop | Cast explicitly at boundaries: `Number(bigintVal)` on serialize, `BigInt(numberVal)` on internalize |
| Coverage < 90% | Ensure all `GameTermination` branches are exercised; use fixture FENs for edge cases |
| `unmakeMove` correctness | Only used in tests/clock-editor per spec; keep simple: reconstruct Chess from `fenHistory[n-1]`, slice `moves` and `fenHistory` by 1. No need to undo clock (test fixture provides explicit clock values). |
