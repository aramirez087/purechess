## Design Decisions

### Brand accent: warm gold
`hsl(43 96% 56%)` — chess-appropriate, avoids conflict with board green used for legal move highlights in later sessions.

### Dark mode mechanism: `[data-theme="dark"]` attribute
`next-themes` with `attribute="data-theme"` sets attribute on `<html>`. CSS uses `[data-theme="dark"]` selector, not `.dark`. This allows programmatic override while defaulting to system preference.

### Fonts: `next/font/google`
- Body: **Geist Sans** (or Inter as fallback) — clean, neutral
- Mono: **JetBrains Mono** — clocks, SAN notation, FEN strings
- Both use `display: 'swap'` automatically via next/font

**Decision: use Geist** (`next/font/local` with `geist` package) for body since it ships with Next.js 14 and avoids network requests in dev. JetBrains Mono via `next/font/google`.

### Toast: sonner
Simpler API than shadcn's radix-based toast. Less boilerplate, native dark mode support.

### shadcn version: `shadcn@latest`
Write `components.json` manually to avoid interactive CLI prompts, then `pnpm dlx shadcn@latest add <component>` for each component in one batch call.

### TypeScript for `components.json`
No — `components.json` is JSON, not TypeScript. shadcn expects it at the workspace root (`apps/web/components.json`).

### Phase 3: `utils.ts`

Write `apps/web/src/lib/utils.ts` before shadcn add, because shadcn components import from `@/lib/utils`.

```typescript
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

export function formatDuration(ms: number): string {
  const totalSeconds = Math.floor(ms / 1000);
  const minutes = Math.floor(totalSeconds / 60);
  const seconds = totalSeconds % 60;
  return `${minutes}:${seconds.toString().padStart(2, '0')}`;
}

export function formatRelativeTime(date: Date): string {
  const diffMs = Date.now() - date.getTime();
  const diffSecs = Math.floor(diffMs / 1000);
  if (diffSecs < 60) return 'just now';
  const diffMins = Math.floor(diffSecs / 60);
  if (diffMins < 60) return `${diffMins}m ago`;
  const diffHours = Math.floor(diffMins / 60);
  if (diffHours < 24) return `${diffHours}h ago`;
  const diffDays = Math.floor(diffHours / 24);
  return `${diffDays}d ago`;
}

export function clampRatingDelta(n: number): number {
  return Math.max(-400, Math.min(400, n));
}
```

## Risks and Mitigations

| Risk | Likelihood | Mitigation |
|------|-----------|------------|
| `pnpm dlx shadcn@latest add` fails interactively or overwrites globals.css | Medium | Pre-write `components.json` fully; use `--overwrite` flag; restore globals.css after if modified |
| shadcn generates `.dark` not `[data-theme="dark"]` in component variants | High | After install, grep `components/ui/**` for `.dark` and replace with `dark:` Tailwind variant (which works with next-themes when `attribute="class"`) OR keep `attribute="class"` in ThemeProvider and change CSS to `.dark {}` |
| Font CSS variable names collision | Low | Use `--font-geist-sans` and `--font-jetbrains-mono` matching next/font conventions |
| `next-themes` flash of wrong theme | Medium | Add `suppressHydrationWarning` to `<html>` in layout.tsx; use `disableTransitionOnChange` |
| TypeScript errors in shadcn components (forwardRef types) | Low | shadcn ships correct types; pin to known-good version if issues arise |
| `geist` package not available | Low | Fall back to `next/font/google` with `Inter`; JetBrains Mono stays on google fonts |

### Critical: dark mode selector conflict

shadcn components use `dark:` Tailwind variant. By default, Tailwind's `dark:` variant triggers on `.dark` class. But we're using `[data-theme="dark"]` attribute.

**Resolution**: In `tailwind.config.ts`, configure `darkMode`:
```typescript
darkMode: ['selector', '[data-theme="dark"]'],
```
This tells Tailwind to activate `dark:` variants when `[data-theme="dark"]` is on an ancestor. All shadcn `dark:` variants work correctly. No need to patch individual components.

This is the correct Tailwind v3.4+ syntax for custom dark mode selectors.
